Goods Insight API 开放平台

开放给第三方集成的 4 个高价值接口。所有请求必须带 HMAC-SHA256 签名头。

1. 鉴权

1.1 请求头

Header必填说明
X-Api-Key是公开的 key_id，形如 gi_live_abc123def456
X-Timestamp是Unix 秒（10 位），与服务端偏差 ≤ 300s
X-Signature是HMAC-SHA256 签名的 hex（小写）

1.2 签名算法

message = METHOD + "\n"
        + REQUEST_PATH + "\n"   # 不含 query string
        + X-Timestamp + "\n"
        + sha256_hex(body)      # GET 请求 body 为空字符串

signature = hex(HMAC-SHA256(secret, message))
⚠️ secret 是创建 API Key 时一次性返回的 40 字符随机串，只显示一次，请妥善保存。 库内不存 secret 原值（生产环境应使用 KMS 加密列）。

1.3 完整示例（curl + openssl）

# 变量
KEY_ID="gi_live_abc123def456"
SECRET="你的40字符secret"
METHOD="POST"
PATH_="/open/v1/profit/estimate"
TS=$(date +%s)
BODY='{"asin":"B0CX12345","marketplace":"US","sellPrice":29.99,"cogs":8.00}'

# 计算签名
BODY_HASH=$(echo -n "$BODY" | openssl dgst -sha256 -hex | awk '{print $2}')
MESSAGE="${METHOD}\n${PATH_}\n${TS}\n${BODY_HASH}"
SIG=$(printf "%b" "$MESSAGE" | openssl dgst -sha256 -hmac "$SECRET" -hex | awk '{print $2}')

# 发请求
curl -X "https://api.goods-insight.com${PATH_}" \
  -H "X-Api-Key: $KEY_ID" \
  -H "X-Timestamp: $TS" \
  -H "X-Signature: $SIG" \
  -H "Content-Type: application/json" \
  -d "$BODY"

2. 接口列表

2.1 POST /open/v1/profit/estimate — 利润估算

输入：ASIN 或 产品 + 售价/成本/费用。输出：净利润、净利率、回本月数、风险等级。

Request:
{
  "asin": "B0CX12345",
  "marketplace": "US",
  "sellPrice": 29.99,
  "cogs": 8.00,
  "fbaFee": 4.50,
  "referralFeePct": 0.15,
  "adCost": 3.00,
  "otherCost": 0.50
}

Response: ProfitEstimationResDTO

2.2 POST /open/v1/risk/assess — 风险评估

输入：ASIN 或 类目 + 售价。输出：总分（0-100）+ 等级（LOW/MEDIUM/HIGH）+ 4 维分项 + AI 建议。

Request:
{
  "asin": "B0CX12345",
  "sellPrice": 29.99
}

Response: RiskScoreResDTO { totalScore, level, breakdown, warnings, aiAdvice }

2.3 POST /open/v1/ideas/generate — 创意生成

输入：材料/工艺/认证/月产能。输出：3-5 个产品创意（按可行性分数降序）。

Request:
{
  "materials": ["Bamboo", "Cotton"],
  "processes": ["Laser Engraving"],
  "monthlyCapacity": 5000,
  "targetPrice": "29.99",
  "ideaCount": 3
}

Response: { count: 3, ideas: [ProductIdeaResDTO, ...] }

2.4 POST /open/v1/trend/forecast — 趋势预测

输入：关键词 + dataType（SEARCH_VOLUME / SALES_VOLUME）。输出：未来 6 个月预测 + AI 建议。

Request:
{
  "keyword": "yoga mat",
  "dataType": "SEARCH_VOLUME",
  "lookbackMonths": 12
}

Response: TrendForecastResDTO

GET 简化版：GET /open/v1/trend/forecast?keyword=yoga+mat

3. 错误码

HTTPcode说明
401missing_credentials3 个必填头缺失
401timestamp_expired时间戳与服务器偏差 > 300s
401invalid_api_keyKey 不存在
403key_not_activeKey 状态非 ACTIVE（已撤销 / 已过期）
403key_expiredKey 超过 expires_at
401invalid_signature签名校验失败（大概率 secret 错）
429qps_exceededQPS 超限
429quota_exceeded本月配额已用完
500—服务端异常（详见 message 字段）

4. 配额规则

  • QPS：单 key 每秒最多 N 次（创建时设，默认 10）
  • 月配额：自然月内最多 N 次（创建时设，默认 10000）。每月 1 日 0 点自动重置
  • 限流策略：先 QPS（每秒窗口），后月配额（DB count）。超限直接 429，不消耗 LLM token
  • 用量查询：GET /api/open/api-keys/{id}/usage?days=7（内部接口，需 JWT）
💡 生产建议：
  1. secret 务必保存在服务端，不要放进前端代码或 App 代码
  2. 请求前先校准本地时间（ntpdate），避免 timestamp_expired
  3. 监听 429 响应并做指数退避
  4. 每分钟最多失败重试 3 次，避免 secret 错配被误判为撞库
